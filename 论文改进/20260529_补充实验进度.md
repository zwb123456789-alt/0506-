## 已完成实验详情

### 实验 1：Phase63 公平消融 ✅

**产物**：`论文改进/补充实验/结果/phase63_ablation/run_20260530_221018/`

**目的**：回应"OCS 用 5 几何，图像只用 phase63，融合是否不公平"。

**结果**（10°→5° split，5 seeds MLP/CNN/Fusion）：

| 方法 | 输入 | mean error | Hit@5° |
|------|------|-----------|--------|
| OCS-only MLP | Phase63 total log 3D | 74.63° | 0.3% |
| OCS-only MLP | Phase63 per_part log 6D | 21.68° | 20.7% |
| OCS-only MLP | Phase63 all log 9D | 23.80° | 20.4% |
| OCS-only MLP | Phase63 all raw 9D | 23.46° | 19.8% |
| **Feature Fusion** | **Phase63 per_part log + Image** | **6.79°** | **63.1%** |
| Feature Fusion | Phase63 total log + Image | 28.68° | 23.8% |

**对比 concat5（已有结果）**：

| 方法 | 输入 | mean error | Hit@5° |
|------|------|-----------|--------|
| OCS-only MLP | concat5 per_part log 30D | 5.91° | 73.8% |
| Feature Fusion | concat5 per_part log + Image | 4.10° | 87.3% |

**结论**：
- 单几何 OCS 确实远弱于多几何（21.68° vs 5.91°），证明多几何 OCS 信息量远超单几何
- 但图像能大幅补偿：Phase63 fusion (6.79°) 接近 concat5 OCS-only (5.91°)
- 比较是公平的，fusion 的增益在单几何下更显著（21.68°→6.79°，提升 69%）

### 实验 2：随机 80/20 Split ✅

**产物**：`论文改进/补充实验/结果/random_split/run_20260530_224054/`（前3个）+ `run_fusion_fix/`（fusion）

**目的**：回应数据量小和 10°→5° split 是否特殊的质疑。

**结果**（random 80/10/10 split，5 seeds）：

| 方法 | 输入 | mean error | Hit@5° |
|------|------|-----------|--------|
| OCS MLP | all_raw 45D | 1.67° | 98.2% |
| OCS MLP | per_part_log 30D | 2.72° | 95.1% |
| CNN image-only | log1p 128×128 | 4.20° | 72.5% |
| Feature Fusion | per_part_log + Image | **2.13°** | **98.6%** |

**对比 10°→5° split**：

| 方法 | 10°→5° mean | Random mean | 差距 |
|------|------------|-------------|------|
| OCS MLP all_raw | 3.98° | 1.67° | 2.4× |
| OCS MLP per_part_log | 5.91° | 2.72° | 2.2× |
| CNN image-only | 12.38° | 4.20° | 2.9× |
| Feature Fusion per_part | 4.10° | 2.13° | 1.9× |

**结论**：
- 10°→5° split 是更严格的泛化测试（需姿态空间插值），论文应以此为主
- Random split 下 fusion 仍优于单模态，互补性结论稳健
- 两者差距 2-3× 是合理的：插值到未见网格点 >> 随机采样已见区域

### 实验 3：BRDF 参数敏感性 ✅

**产物**：`论文改进/补充实验/结果/brdf_sensitivity/run_20260529_220609/`

**目的**：回应"材料参数 nominal、非实测"的问题。

**方法**：GGX 参数 ±20% 扰动，phase63 几何，30° 网格（91 姿态）

**结果**（OCS 相对变化，按影响排序）：

| 参数 | 部件 | ±20% 时 OCS 中位变化 | 最大变化 |
|------|------|---------------------|---------|
| **roughness** | jinshuzhuti (金属) | **~30-42%** | **46.2%** |
| **F0** | jinshuzhuti (金属) | ~13-16% | 16.5% |
| base_color | taiyangnengban (太阳能板) | ~4% | 6.5% |
| base_color | yinshenban (遮光板) | ~1.7% | 3.1% |
| roughness | taiyangnengban | <1% | 0.9% |
| roughness | yinshenban | <0.2% | 0.4% |
| **base_color** | **jinshuzhuti** | **0%** | **0%** |

**关键发现**：
- 金属主体 roughness 是最敏感参数（决定镜面峰宽度），但 ±20% 扰动合理（材料公差范围）
- 金属 base_color 零影响：metallic=1 完全压制漫射项，符合 GGX 物理
- 非金属部件（太阳能板/遮光板）参数变化影响 <5%，几乎不敏感
- 论文表述："BRDF 参数在合理范围内扰动对 OCS 影响可控，金属 roughness 是最关键的待标定参数"

### 实验 4：遮挡 w/ vs w/o 对比整理 ✅

**产物**：`论文改进/补充实验/结果/occlusion_analysis/run_20260529_215456/`

**目的**：证明自遮挡不是装饰模块。

**结果**（5 几何 GGX，2701 姿态）：

| Geometry | Phase | OccRatio Mean | OccRatio Max |
|---|---|---|---|
| phase63_backscatter | 63° | 69.7% | 97.1% |
| phase24_near_backscatter | 24° | 60.1% | 97.7% |
| phase120_forward_scatter | 120° | **78.5%** | 99.9% |
| phase90_side | 90° | 72.6% | 99.5% |
| phase45_overhead | 45° | 68.4% | 96.6% |

**分部件事后遮挡率 (mean)**：

| Part | phase63 | phase120 |
|------|---------|----------|
| jinshuzhuti (金属主体) | 75.8% | 80.7% |
| taiyangnengban (太阳能板) | 65.5% | 55.5% |
| yinshenban (遮光板) | 40.3% | 59.8% |

**结论**：自遮挡率跨几何 60-78.5%，金属主体最高。大相位角（120°）遮挡最严重。自遮挡对 OCS 有实质性影响，不是装饰模块。

### 实验 7：Roll 小规模敏感性 ✅

**产物**：`论文改进/补充实验/结果/roll_sensitivity/run_20260529_221408/`

**目的**：评估固定 roll=0 假设的合理性。

**方法**：7 代表姿态 × 7 roll 角（0°~180°），phase63 GGX

**结果**：

| yaw | pitch | OCS(roll=0) | max rel change |
|-----|-------|-------------|---------------|
| 150° | -80° | 0.0131 | **26.2%** |
| 270° | -45° | 0.0129 | **26.0%** |
| 0° | -45° | 0.0116 | 22.5% |
| 90° | -45° | 0.0116 | 22.4% |
| 45° | 0° | 0.0101 | 16.2% |
| 180° | 0° | 0.0105 | 15.6% |
| 0° | 0° | 0.0099 | 13.4% |

**Overall：mean rel_range = 20.3%，max = 26.2%**

**结论**：Roll 变化导致 OCS 平均约 20% 变化，极端姿态近 26%。论文可将此作为 limitation："本文聚焦固定 roll 的两轴反演；roll 敏感性表明框架可扩展到三轴，但完整 3-DOF 数据集约需 37 倍计算量，留作后续工作。"

## 遇到的 Bug 及修复记录

### Bug 1：`OBS_GEOMETRIES` 数据结构
- `config.py` 中为 `(sun, det, label)` 元组，非 dict
- 修复：`g[0]`/`g[1]`/`g[2]` 替代 `g["sun"]`/`g["det"]`/`g["label"]`

### Bug 2：`fast_simplification` 缺失
- trimesh simplify 需要 C++ 后端，未安装
- 修复：移除 simplify 调用，直接使用完整 mesh

### Bug 3：`RayForest` API 不匹配
- `RayForest(verts, faces)` vs `RayForest(meshes_dict)`
- 修复：移除 RayForest 构建，敏感性分析不需要遮挡计算

### Bug 4：脚本迁移后 `sys.path` 失效
- 从 `03_inversion/` 迁移到 `论文改进/补充实验/代码/` 后 import 路径错误
- 修复：`_PROJECT_ROOT` 多加一层 `..`，添加 `ocs_project/03_inversion/` 到 sys.path

### Bug 5：Fusion 训练 batch 格式不兼容
- fusion DataLoader 返回 (img, ocs, y) 三元组，但 `train_epoch` 期望 (X, y) 二元组
- 修复：`train_epoch`/`evaluate` 改为条件判断 `if len(batch) == 2`
