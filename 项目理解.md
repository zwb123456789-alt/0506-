# 3.遮挡验证（已完成）

> 本节汇总 2026-05-12 两轮修复 + 完整验证的最终结论。历史问题描述保留在文末的"历史背景"子节。

## 3.1 核心结论

- **遮挡机制**：从旧版 `exclude_parts={当前部件}` 切换为 **`min_hit_distance` 过滤射线起点自相交**（保留当前部件参与命中判定，按距离过滤）。
- **阈值决策**：**`EPSILON = 1.0 mm`**（config.py 默认值）—— 经真实三件套 `min_hit_distance` 敏感性扫描验证，是三部件共同的安全平坦段上限。
- **代码状态**：`occlusion.py` 已修；`04_tests/test_occlusion_geometry.py` 与 `05_occlusion_validation/run_occlusion_validation.py` 已重写对齐新 API。
- **验证产物**：`结果/遮挡验证/run_20260512_213850/`（最新）。

## 3.2 当前遮挡逻辑

```python
# ocs_core.py 中调用
occ_sun, occ_det = ray_forest.batch_occlusion_dual(
    origins_M, sun_dir_M, det_dir_M, min_hit_distance=EPSILON)
```

含义：

- **不排除任何部件**，所有 STL 都参与命中判定
- 射线起点 = 面元中心 + 法向 × `EPSILON`（已偏移）
- 命中后保留 `dist > min_hit_distance` 的命中点，距离更近的命中（=自相交）被过滤
- 同部件内部**真实距离 > 1 mm** 的遮挡能被正确检出（修复了 U 型块 / 凹腔类几何的漏检）

## 3.3 为什么切换机制

| 旧逻辑 `exclude_parts={当前部件}` | 新逻辑 `min_hit_distance=EPSILON` |
|---|---|
| 把整块部件从 BVH 中踢掉 | 所有部件都参与，用距离过滤 |
| 优点：杜绝自相交 | 优点：同部件内部真实遮挡能检出 |
| 缺点：U 型、凹腔、加强筋间互遮挡**全部漏检** | 缺点：阈值选错会误杀 1~5 mm 近邻面遮挡 |

## 3.5 验证结果（run_20260512_213850）

### 合成场景

**17 / 17 PASS**。

关键项：

- 单平板所有 `(epsilon > 0, mhd > 0)` 组合 hit_ratio = 0 → 自相交彻底压住
- 双平板跨部件 hit_ratio = 1.0
- **U 型块同部件内部 hit_ratio = 1.0**（修复核心）
- 嵌套圆柱 hit_ratio = 1.0

|---:|---:|---:|---:|---:|---:|---:|

- `taiyangnengban` 平坦段到 ~1 mm
- `yinshenban` 平坦段到 ~2 mm
- **三部件共同的安全上限 ≈ 1.0 mm** → `EPSILON = 1.0 mm` 是当前最优选择

## 3.6 遗留问题

1. **`ocs_no_occ` 保持不变，但 `ocs_with_occ` mean 遮挡率 = 72.88%**（相较旧版 35.78% 增 +37pp）。主要贡献来自 `jinshuzhuti` 的真实几何（内部凸台/加强筋），**不是阈值误杀**。
2. **意外发现**：合成验证 `epsilon=0 + mhd=0` 实测 hit_ratio = 0（预期 1.0 即暴露自相交）。原因：Embree `intersects_location` 内置 face-self 跳过，起点恰在面上时不把源面当命中。此项标记为 `expose_self` 不判，不影响 PASS。
3. **未来可选（非当前队列）**：若仍需降低 jinshuzhuti 遮挡率，可改 `occlusion.py` 令 `batch_occlusion_dual` 返回命中面所属部件，对"同部件命中"用较大 mhd、对"跨部件命中"用 1 mm。改动大且可能无法区分"真实同部件遮挡"与"几何粘连"，需先确认 72.88% 确实是问题再做。
