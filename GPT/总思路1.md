# 七、你当前代码的一个重要逻辑提醒

你的遮挡检测中有：

```python
occ_sun, occ_det = ray_forest.batch_occlusion_dual(
    origins_M, sun_dir_M, det_dir_M, exclude_parts={mat_name})
```

这里 `exclude_parts={mat_name}` 表示：

> 检测遮挡时排除当前部件，只检测其他部件是否遮挡。

这个策略有利于避免自相交，但也会忽略 **同一部件内部的自遮挡**。

例如：

```text
金属主体自身凹陷结构遮挡金属主体其他面
```

在当前代码中可能不会被检测到。

短期为了跑通流程可以保留。  
但后续科研验证时，建议改成：

```text
不排除当前部件，但通过射线起点 epsilon 和 hit distance 排除自身三角面。
```

这属于后续“遮挡验证”阶段，不建议现在马上改。

## 反对理由 1：继续维护当前大脚本会越来越难

## 反对理由 2：当前遮挡算法排除当前部件，物理上不够严格

当前：

```python
exclude_parts={mat_name}
```

会导致同一部件内部自遮挡被忽略。

对于复杂金属主体，如果有凹槽、舱体突起、折叠结构，这可能低估遮挡。

### 反方会如何决策？

他们会要求先完成严格遮挡验证：

```text

双平板
U 形结构

```

然后再跑姿态扫描。

你安装的是 Blender 5.0。
