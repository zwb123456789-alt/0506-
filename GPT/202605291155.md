# 9. 你现在论文的核心结果应该怎么讲

你现在有一个潜在问题：

```text
OCS MLP all_raw = 3.98°
Feature fusion per_part_log = 4.10°
```

审稿人可能问：

> 既然 OCS-only 已经 3.98°，为什么还需要融合？

你的回答不能是“融合最好”，因为它不是全局最好。  
正确叙事是：

```text
all_raw 45D contains semi-oracle information, including rich part-level and occlusion-related features. It represents an upper-bound OCS-only case.
```

然后强调：

```text
In the more realistic medium-information setting, per_part_log, feature-level fusion improves the OCS-only baseline from 5.91° to 4.10°, corresponding to a 31% reduction in mean angular error.
```

也就是说：

```text
all_raw = upper bound
per_part_log = proposed practical setting
total_log = weak OCS setting
```

论文主张不要写成：

> Fusion always outperforms OCS.

而要写成：

> Fusion is most beneficial when the OCS modality is informative but not over-dominant. This reveals a modality-strength-dependent complementarity.

这个说法更高级，也更诚实。
