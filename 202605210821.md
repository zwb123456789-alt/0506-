同意：**下一步先做 OCS-only MLP 连续回归是合理的**。它正好补 kNN/NN 检索“只能返回库内姿态、不能输出 5° 插值点”的短板，而且不依赖新版图像渲染。

但建议你在 `train_mlp.py` 里做几处小调整，避免后面结果解释有坑。

## 2. 先修正 kNN 对比基线的表述

你现在说：

> kNN 无法姿态插值，10°→5° Top1@5° < 1.1%

这个结论方向对，但建议更严谨地写成：

> **离散 NN/kNN 检索不能输出训练库外姿态，只能返回最近的 10° 网格姿态；因此它在 5° 插值任务上存在天然量化误差。**

另外，之前 `median=5.00°` 但 `Top1@5°=1.10%` 很像是用了严格 `<5°`。建议基线脚本统一改成：

```python
hit5 = err <= 5.0 + 1e-6
hit10 = err <= 10.0 + 1e-6
```

否则 MLP 和 kNN 对比会不公平。

---

## 3. 特征模式建议：一定加一个真实可观测 5D

你说：

```text
45D all / 15D total / 30D per_part
```

这个可以跑，但建议额外加一个：

```text
obs_total_5D = 每个几何只取最终观测总 OCS，共 5 维
```

原因是：

- `45D all` 大概率包含分部件、无遮挡、遮挡损失等仿真内部量；
- `15D total` 如果包含 `no_occ / with_occ / loss`，也不是纯真实观测；
- 真实探测器最直接拿到的是每个观测几何的总 OCS。

所以建议特征预设这样命名：

| preset | 维度 | 定位 |
|---|---:|---|
| `obs_total` | 5D | 最接近真实观测，论文主线必须有 |
| `total` | 15D | 总量级 oracle，如果含 no_occ/loss 要说明 |
| `per_part` | 30D | 部件级 oracle / 诊断 |
| `all` | 45D | 仿真全特征 upper bound |

论文里主结论最好不要只依赖 `all45`。

## 4. 输入预处理建议

GGX OCS 动态范围很大，MLP 直接吃 raw 容易被 specular peak 主导。建议默认用：

```python
X_ocs = np.log10(np.clip(X_ocs, eps, None))
X = StandardScaler().fit(X_train).transform(X)
```

注意：

1. scaler 只能在 train 上 fit；
2. log 只用于 OCS / loss 这类非负量；
3. 遮挡率这类 `[0,1]` 特征不要 log，直接 z-score；
4. 建议保留 `--transform raw/log` 做消融，但 MLP 主结果优先看 log。

## 5. 输出编码和解码

目标编码：

```python
def encode_target(yaw_deg, pitch_deg):
    y = np.deg2rad(yaw_deg % 360.0)
    p = np.deg2rad(pitch_deg)
    return np.stack([
        np.sin(y), np.cos(y),
        np.sin(p), np.cos(p)
    ], axis=1)
```

预测解码时建议先归一化 sin/cos 对：

```python
def decode_pred(pred):
    ys, yc, ps, pc = pred.T

    yr = np.sqrt(ys**2 + yc**2) + 1e-8
    pr = np.sqrt(ps**2 + pc**2) + 1e-8

    ys, yc = ys / yr, yc / yr
    ps, pc = ps / pr, pc / pr

    yaw = (np.rad2deg(np.arctan2(ys, yc)) + 360.0) % 360.0
    pitch = np.rad2deg(np.arctan2(ps, pc))
    pitch = np.clip(pitch, -90.0, 90.0)

    return yaw, pitch
```

Loss 可以先用普通 MSE：

```python
loss = mse(pred, target)
```

可选加一个 unit norm 正则：

```python
norm_loss = ((ys**2 + yc**2 - 1)**2).mean() + ((ps**2 + pc**2 - 1)**2).mean()
loss = mse_loss + 0.01 * norm_loss
```

## 6. MLP 结构建议

703 个训练样本不多，不建议网络太大。初版可以：

```text
Input dim: 5 / 15 / 30 / 45
Hidden: 128 → 128 → 64
Activation: SiLU or ReLU
Norm: BatchNorm1d 或 LayerNorm
Dropout: 0.05 ~ 0.15
Output: 4
Optimizer: AdamW
LR: 1e-3
Weight decay: 1e-4
Batch size: 64
Epoch max: 1000~2000
Early stopping patience: 100~200
```

如果 BN 在小 batch 下不稳定，可以换 `LayerNorm`。

## 9. 建议额外加一个很便宜的连续基线

为了证明 MLP 的优势不是单纯来自“输出连续值”，建议加一个：

```text
distance-weighted KNN regressor
```

即 kNN 不返回最近姿态，而是对邻居的 `[sin yaw, cos yaw, sin pitch, cos pitch]` 做距离加权平均。

例如：

```python
KNeighborsRegressor(
    n_neighbors=5,
    weights="distance",
    metric="euclidean"
)
```

这会是一个很好的传统连续回归 baseline。

对比关系可以是：

```text
NN discrete retrieval
KNN weighted regression
MLP regression
```

这样论文逻辑会更完整。

## 结论

可以进入 MLP。建议 `train_mlp.py` 第一版就按这个优先级实现：

1. `obs_total / total / per_part / all` 特征预设；
2. `log + zscore`，scaler 只 fit train；
3. 10°→5° split；
4. sin/cos 输出；
5. 5 seeds；
6. 同时补一个 weighted-kNN regression baseline；
7. 修正 Hit@5° 的 `<= threshold + eps`。

这样 Step 11c 的结果会非常清楚：  
**离散检索 vs 连续回归，真实总 OCS vs oracle 多特征，多观测几何下 OCS-only 到底能插值到什么精度。**
