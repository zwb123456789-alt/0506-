# 顶刊实验体系总方案

> 面向项目：OCS + 图像联合仿真 / 姿态反演 / 观测几何优化  
> 当前基线：真实卫星模型、暗室条件、真实太空照片（无姿态真值）、现有 OCS/BRDF/遮挡/反演代码  
> 写作目标：为顶刊和 SCI 一区论文建立可执行、可扩展、可复现实验体系

---

## 0. 先确认项目与论文方向

你的项目不是“做一个卫星渲染器”，而是要建立一套可用于空间目标研究的统一物理链路：

```text
真实几何
→ 真实或可标定材料
→ BRDF
→ 自遮挡/多部件耦合
→ OCS / 光度图像
→ 姿态反演
→ 观测几何优化
→ 真实数据验证
```

适合拆成多篇论文，而不是硬塞进一篇：

1. 物理前向模型与实验验证
2. OCS-图像联合姿态反演
3. 观测几何与可观测性分析
4. Sim2Real 与真实太空照片弱监督验证

## 1. 顶刊审稿人会先看什么

### 1.1 期刊视角

| 期刊 | 审稿人核心关注 |
|---|---|
| Nature Astronomy | 是否有跨领域意义，是否不仅是工程结果，而是能回答更广泛的天文/空间目标观测问题 |
| TGRS | 方法是否新，实验是否完整，实验条件、装置、方法、数据是否可复现 |
| TAES | 是否体现复杂系统、空间系统、地基系统的工程闭环和系统级贡献 |
| Acta Astronautica | 是否和空间技术、Earth-based systems、观测与任务应用紧密相关 |
| Remote Sensing | 是否有 validation/calibration，方法是否可复现，数据是否规范 |
| Optics Express | 光学/光子学创新是否是核心，是否有明显 novelty 和 impact，是否只是增量工作 |

### 1.2 顶刊审稿人最常追问的问题

1. 你的 BRDF 参数从哪里来？
2. 你的自遮挡模型是否经过独立验证？
3. 你的 OCS 和图像是否真的共享同一套前向物理？
4. 你的结果是否只在仿真里成立？
5. 你的姿态反演是否有真值？
6. 多观测几何到底提高了多少可观测性？
7. 实验结果是否对材料、曝光、噪声、域差敏感？
8. 你的结论是否只对单一模型或单一场景成立？
9. 是否有重复实验和误差条？
10. 是否有真实数据或独立验证链路？

如果这些问题答不严密，顶刊审稿通常会把工作判成“工程实现不错，但科学贡献不足”。

## 4. 实验规范总原则

### 4.1 必须有三类真值

1. **材料真值**：材质样片、表面处理、几何尺寸、粗糙度来源。
2. **姿态真值**：yaw/pitch/roll、角速度、重复测量误差。
3. **光度真值**：曝光、增益、响应线性、光源方向、距离、背景。

### 4.2 必须有四类控制

1. 控制光源方向和稳定性。
2. 控制相机链路。
3. 控制姿态与重复采样。
4. 控制背景、噪声、遮挡和曝光。

### 4.3 每个实验都要留四份结果

1. 原始数据。
2. 标定与元数据。
3. 处理后的结果。
4. 失败样本与异常样本。

### 4.4 每个实验都要有重复

至少做：

- 同姿态重复 3 次
- 同天重复
- 跨天重复

顶刊很看重 repeatability，不只是单次好看。

## 5. 详细实验方案

### E0. 相机与光源标定实验

#### 目的

把“图像”变成可测量量，而不是只是一张 JPG/PNG。

#### 顶刊审稿人会问

- 你的灰度是否线性？
- 你的曝光是否饱和？
- 你的相机响应曲线是否已知？
- 背景和暗电流怎么处理？

#### 材料与技术要求

- 目标相机 1 台，最好是可输出 RAW 或线性灰度的工业相机/科学相机
- 稳定均匀光源
- 标准白板或漫反射参考板
- 暗场可控环境
- 记录曝光、增益、白平衡、焦距、光圈、温度

#### 实施步骤

1. 采集暗场帧，测暗电流与噪声底。
2. 采集平场帧，测视场不均匀和镜头暗角。
3. 做曝光线性扫描，检查灰度-曝光是否线性。
4. 做增益扫描，确认可用动态范围。
5. 测 PSF 或点光源响应，估计模糊核。
6. 固化标定版本，后续所有实验统一引用。

#### 需要获取的数据

- 暗场图像
- 平场图像
- 曝光线性曲线
- 增益曲线
- PSF/模糊核
- 响应函数

#### 输出指标

- 线性度 R²
- 噪声底
- 动态范围
- 平场均匀性

#### 论文支撑

- 所有图像实验
- Optics Express / Remote Sensing / TGRS

### E1. 材料 BRDF 实测

#### 目的

给你的 GGX / Cook-Torrance 参数一个现实依据，而不是只用文献 nominal 值。

#### 顶刊审稿人会问

- 你的 roughness、F0、base color 从哪里来？
- 不同材料是否真的有不同散射特性？
- 空间材料老化后参数会不会变？

#### 建议材料

1. 金属主体样片：铝合金、镀铝片、金属喷涂件
2. 太阳能板样片：太阳能电池片、玻璃盖片、深蓝半导体表面
3. 遮光/低反射样片：黑涂层、黑 Kapton、消光黑表面
4. 可选参考样片：标准漫反射白板、镜面片

#### 技术要求

- 角度可控的光源与探测器几何
- 已知入射/出射方向
- 波段可控，至少做 450/550/650/850 nm 或宽谱 400-1000 nm
- 若可能，加入偏振测量
- 样片表面平整度和尺寸要记录

#### 实施步骤

1. 样片编号并测量尺寸、粗糙度、表面处理方式。
2. 先对参考板做系统标定。
3. 按入射角、观测角、波长扫描测量散射强度。
4. 每个角度至少重复 3 次。
5. 拟合 GGX / Cook-Torrance 参数。
6. 比较不同材料的参数差异与残差。
7. 如果条件允许，做老化前后对比。

#### 需要获取的数据

- 入射角、出射角、波长
- 原始光强/反射强度
- 背景强度
- 重复测量值
- 样片信息
- 温度、时间、仪器状态

#### 输出指标

- BRDF 曲线
- 拟合参数
- 拟合 RMSE
- 重复性标准差
- 参数置信区间

#### 论文支撑

- BRDF 模型论文
- 材料不确定性论文
- Optics Express / TGRS / Acta Astronautica

#### 改进建议

- 不要只测单个角度。
- 不要只测单个波段。
- 不要只测一次。
- 最好对比“实测参数 vs 文献参数 vs 仿真假设参数”。

### E2. 简单基准几何验证

#### 目的

先验证最简单的几何体是否能被你的物理链路正确解释，再上真实卫星模型。

#### 推荐几何

1. 平板
2. L 型板
3. U 型块
4. 双板遮挡结构
5. 圆柱 / 嵌套圆柱

#### 顶刊审稿人会问

- 你的遮挡逻辑在简单几何上都对吗？
- 你的模型是不是只对复杂对象“看起来合理”？
- 你能否和解析预期或独立基准一致？

#### 技术要求

- 可快速制作的标准几何件
- 清楚的尺寸标定
- 可重复的姿态控制
- 一致的光源与相机链路

#### 实施步骤

1. 制作标准几何件并记录精确尺寸。
2. 在暗室内以固定光源和相机观测。
3. 做单几何、双几何和遮挡配置实验。
4. 按 yaw/pitch 采样扫描。
5. 与代码中的模拟结果逐姿态对比。
6. 特别检查遮挡边界、峰值位置和消失区。

#### 需要获取的数据

- 几何尺寸
- 姿态真值
- 图像
- 积分亮度
- 预期遮挡关系

#### 输出指标

- OCS 曲线形状是否一致
- 遮挡率是否合理
- 峰值角是否匹配
- 误差热图

#### 论文支撑

- 方法验证
- 遮挡模型验证
- 适合作为所有主论文的前置 sanity check

### E3. 部件级散射与遮挡实验

#### 目的

证明三部件模型不是形式上拼出来的，而是物理上真的有可分离贡献。

#### 顶刊审稿人会问

- 主体、太阳翼、遮光板的贡献是否真的不同？
- 遮挡是否主要发生在部件之间还是部件内部？
- 部件级参数是否可识别？

#### 材料与技术要求

- 真实卫星模型
- 可拆卸部件或可遮挡部件
- 可记录部件安装状态
- 同步记录每个部件是否可见

#### 实施步骤

1. 单部件观测：只保留主体、只保留太阳翼、只保留遮光板。
2. 双部件观测：主体+太阳翼、主体+遮光板、太阳翼+遮光板。
3. 全部件观测：完整模型。
4. 在相同姿态和光照下重复测量。
5. 比较各部件贡献和遮挡变化。

#### 需要获取的数据

- 部件装配状态
- 每个姿态下各部件图像
- 各部件积分亮度
- 遮挡关系标注

#### 输出指标

- 部件级 OCS 贡献
- 部件级遮挡率
- 部件贡献差异图

#### 论文支撑

- 多材料模型
- 遮挡模型
- 部件级消融

### E4. 全模型暗室 OCS 实验

#### 目的

证明你的 OCS 前向模型在真实模型上成立。

#### 顶刊审稿人会问

- 你的 OCS 不是只在仿真里自洽吗？
- 真实模型的积分亮度是否跟理论一致？
- 你能否解释亮度峰值和凹谷？

#### 技术要求

- 真实卫星模型
- 已标定相机或光度计
- 可控光源
- 姿态控制台
- 黑背景
- 稳定电源和遮光条件

#### 实施步骤

1. 固定光源-相机几何，建立基准坐标系。
2. 对真实模型做零位校准。
3. 扫 yaw / pitch / roll。
4. 记录每个姿态下总亮度或积分灰度。
5. 重复采样，估计方差。
6. 与 OCS 仿真逐姿态对齐。

#### 需要获取的数据

- 姿态真值
- 光源方向
- 相机方向
- 距离
- 积分亮度
- RAW 图像

#### 输出指标

- 实测 OCS 曲线
- 与仿真曲线相关系数
- NRMSE
- 峰值角误差
- 动态范围

#### 论文支撑

- 主论文前向模型
- 真实模型验证
- OCS 反演

#### 改进建议

- 每个关键姿态点至少重复 3 次
- 同一模型在不同日期重复一次
- 随机打乱姿态顺序，避免时间漂移

### E5. 全模型暗室图像实验

#### 目的

证明你的图像渲染和真实成像趋势一致。

#### 顶刊审稿人会问

- 你的图像只是“像”，还是能定量匹配？
- 亮区、阴影、轮廓、glint 是否一致？
- 图像是被曝光和后处理塑形出来的吗？

#### 技术要求

- 高分辨率相机
- RAW 输出优先
- 稳定对焦
- 可做 HDR 或多曝光
- 低反光黑背景

#### 实施步骤

1. 固定同一组姿态和光照几何。
2. 采集单曝光和多曝光图像。
3. 保留 RAW 与处理后版本。
4. 标注亮区、阴影、轮廓、glint。
5. 与渲染图逐姿态对比。

#### 需要获取的数据

- 图像原件
- 曝光/增益/光圈
- 相机内参
- 轮廓标注
- 亮区位置

#### 输出指标

- SSIM
- 轮廓 IoU
- 亮区重心误差
- 灰度直方图差异
- glint 位置误差

#### 论文支撑

- 图像渲染论文
- Sim2Real 论文
- 联合反演论文

### E6. OCS + 图像联合反演实验

#### 目的

证明联合模态优于单模态。

#### 顶刊审稿人会问

- OCS-only、image-only、joint 三者差多少？
- 改善是不是统计显著？
- 是否对新姿态、新几何也有效？

#### 技术要求

- 来自 E4/E5 的带真值数据
- 数据拆分策略
- 统一评价脚本

#### 实施步骤

1. 以姿态真值为标签，构造训练/验证/测试集。
2. 做 LOO、10°→5°、随机 split 三类评估。
3. 对比 OCS-only、image-only、OCS+image。
4. 对比 kNN、MLP、传统匹配、可选 CNN。
5. 输出错误案例和失败模式。

#### 需要获取的数据

- 每个样本的姿态真值
- OCS 特征
- 图像特征
- 联合特征
- 模型输出与误差

#### 输出指标

- mean / median / P90 angular error
- Hit@5°
- Hit@10°
- Top-k retrieval
- 失败样本类型

#### 论文支撑

- 主反演论文
- 联合模态论文

### E7. 多观测几何与可观测性实验

#### 目的

回答“在哪些观测几何下，姿态最容易被辨识”。

#### 顶刊审稿人会问

- 你为什么选这些太阳/探测器方向？
- 你的几何选择是经验的还是可证明的？
- 你是否能给出最优观测策略？

#### 建议几何

```text
phase 24 deg   近后向散射
phase 45 deg   中等相位角
phase 63 deg   你的主基线
phase 90 deg   侧向观测
phase 120 deg  大相位角/前向散射
```

#### 技术要求

- 可移动光源
- 可移动相机或固定相机加可变光源
- 几何角度可精确测量
- 每个几何都要独立标定

#### 实施步骤

1. 固定姿态网格。
2. 改变 phase angle 或等效光照-观测几何。
3. 采集全套图像和光度数据。
4. 对每个几何单独做反演。
5. 比较误差与可分性。
6. 选择最优几何组合。

#### 需要获取的数据

- phase angle
- 光源方向
- 相机方向
- 姿态真值
- OCS/图像输出

#### 输出指标

- 不同几何下的姿态误差
- 可分性指标
- 几何消融结果
- 最优观测区间

#### 论文支撑

- 可观测性论文
- 观测几何优化论文

#### 改进建议

- 不是只做一个几何。
- 至少 3 组，最好 5 组。
- 每组都要单独给误差条。

### E8. Glint 细扫实验

#### 目的

把镜面峰从“麻烦”变成“信息源”。

#### 顶刊审稿人会问

- 高光峰是噪声还是约束？
- 峰值是否重复稳定？
- 粗糙度变化会不会改变峰宽？

#### 技术要求

- 高动态范围采集
- 高角分辨率控制
- 稳定光源
- 低曝光饱和
- 细步长姿态扫描

#### 实施步骤

1. 找到理论上会出现 glint 的姿态附近。
2. 用 0.5°~2° 步长细扫。
3. 记录峰值强度、峰宽、峰位。
4. 重复多次确认稳定性。
5. 对比不同材料/表面处理下的峰形差异。

#### 需要获取的数据

- 高角分辨率光度曲线
- 峰值位置
- 峰宽和半高宽
- 是否饱和
- 材料参数

#### 输出指标

- glint angle error
- peak width
- peak repeatability
- peak-to-background ratio

#### 论文支撑

- 镜面峰约束论文
- 高光指数敏感性论文

### E9. 遮挡消融实验

#### 目的

证明自遮挡不是可有可无，而是决定 OCS 和图像结构的核心因素。

#### 顶刊审稿人会问

- 如果不考虑遮挡，你的结论会差多少？
- 遮挡误差主要来自哪里？
- 你的模型是否能识别同部件内部遮挡和跨部件遮挡？

#### 技术要求

- 可遮挡/可拆卸部件
- 可加入临时遮挡片
- 同样姿态、同样光照、同样相机链路

#### 实施步骤

1. 全模型无改动采样。
2. 人为增加或移除某个部件的遮挡关系。
3. 构造“无遮挡 / 部件遮挡 / 同部件遮挡 / 跨部件遮挡”四类场景。
4. 比较 OCS、图像和反演误差。

#### 需要获取的数据

- 遮挡关系记录
- 图像
- OCS
- 实测峰值/亮度变化

#### 输出指标

- 遮挡率变化
- 反演误差变化
- 部件可见性变化

#### 论文支撑

- 遮挡模型论文
- 方法消融论文

### E10. 传感器链路鲁棒性实验

#### 目的

回答你的方法是否只在理想采集下有效。

#### 顶刊审稿人会问

- 你的方法对噪声、模糊、曝光、背景光敏感吗？
- 在饱和和欠曝条件下还能工作吗？
- 你的结果是否被后处理强行美化？

#### 技术要求

- 可调曝光/增益
- 可调焦距或离焦
- 可人为加入背景和噪声
- 可做多曝光/HDR

#### 实施步骤

1. 固定姿态和光照。
2. 改变曝光、增益、焦点、背景亮度。
3. 记录不同噪声条件下的图像和反演结果。
4. 做从“理想”到“恶劣”的梯度实验。

#### 需要获取的数据

- 曝光、增益、焦距
- 背景亮度
- SNR 估计
- 图像质量指标

#### 输出指标

- 误差 vs SNR 曲线
- 误差 vs blur 曲线
- 误差 vs 背景光曲线
- HDR 前后差异

#### 论文支撑

- Robustness
- Sim2Real
- Engineering validation

### E11. 户外长距离实验

#### 目的

把暗室结果推到更接近真实观测的环境。

#### 顶刊审稿人会问

- 你的方法在真实空气扰动、背景变化、长距离条件下还成立吗？
- 它是否只是暗室方法？

#### 技术要求

- 远距离观测位置
- 目标模型可控姿态
- 长焦相机或望远镜
- 记录气象和距离信息

#### 实施步骤

1. 在较远距离布置目标和观测设备。
2. 扫描姿态与几何。
3. 记录大气、背景、距离、时间。
4. 与暗室结果和仿真结果对比。

#### 需要获取的数据

- 距离
- 天气/气象
- 目标角尺度
- 图像
- 光度序列

#### 输出指标

- 远距离图像质量
- 相对亮度变化
- 与暗室的偏差

#### 论文支撑

- Sim2Real
- 外场验证

### E12. 真实太空照片弱监督实验

#### 目的

利用你已经拿到的真实卫星照片，即使没有姿态真值，也能做真实域验证。

#### 顶刊审稿人会问

- 没有姿态真值，你怎么证明方法真的对真实卫星有效？
- 真实照片和仿真图像之间的差异到底是什么？
- 你有没有做弱监督或候选姿态检索？

#### 技术要求

- 真实卫星照片原始版本
- 拍摄时间、曝光、传感器信息
- 若可能，获取轨道、相位角或大致观测方向

#### 可行做法

1. 用仿真库生成大量候选姿态图像。
2. 对真实照片做检索，返回 Top-k 候选姿态。
3. 比较轮廓、亮区、glint 和灰度分布。
4. 用时序连续性约束候选姿态跳变。
5. 如果有同一目标多张照片，做序列一致性验证。

#### 需要获取的数据

- 真实照片原件
- 观测时间
- 图像处理版本
- 候选仿真库
- 匹配分数

#### 输出指标

- Top-k 一致性
- 相似度分数
- 候选姿态分布宽度
- 弱监督稳定性

#### 论文支撑

- Sim2Real 论文
- 真实域外推论文

#### 关键提醒

真实照片没有姿态真值，所以**不要**把它当作“姿态误差”证据。  
它的正确用途是：

1. 证明你的模型能进入真实域。
2. 证明真实照片能被仿真库解释。
3. 证明真实域和仿真域的差异可以被分析。

## 10. 对当前方案的具体改进建议

1. 不要只做一个统一大实验，要做“校准 - 基准 - 主模型 - 外场 - 弱监督”的层级链。
2. 不要只做真实模型整机实验，要补简单几何基准件。
3. 不要只做单一 phase angle，要做至少 3 组，最好 5 组。
4. 不要只记录图像，要同步记录光度、姿态、环境和标定版本。
5. 不要只看平均值，要给误差条、重复性、失败样本。
6. 不要只用一套材料参数，要做实测、文献值和扰动值对比。
7. 不要只证明“能跑”，要证明“为什么对、对到什么程度、什么条件下会失败”。
8. 不要把真实太空照片当作姿态真值；它的用途是外推验证和弱监督。
